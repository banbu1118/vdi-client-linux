# VDI Client 编译安装教程

基于 **Qt 6 + FreeRDP 3.28.0** 的 VDI 远程桌面客户端编译指南。

---

## 项目结构

```
/home/kk/vdi-client/VDIClient/
├── freerdp-3.28.0/          # FreeRDP 源码 + 编译产物
│   ├── build/               # CMake 构建目录
│   └── install/             # 安装目录（头文件+库）
├── qfreerdp-master/         # qf-client (QML + FreeRDP)
│   └── .codex/              # 文档
├── vdi-client-linux/        # VDIClient (Qt Widgets)
│   ├── src/
│   │   ├── main.cpp
│   │   ├── loginwindow.cpp  # 登录、VM 管理主窗口
│   │   └── loginwindow.h
│   └── build/bin/           # 构建产物 (VDIClient + qf-client)
└── vdi-client-windows-main/ # Windows 版 VDIClient (C#)
```

**qdient 架构**：VDIClient (QWidgets) 负责登录/VM管理，通过 QProcess 启动 `qf-client` (QML+FreeRDP) 作为子进程。

```
VDIClient (QWidgets)              qf-client (QML + FreeRDP)
┌──────────────────────┐          ┌─────────────────────────┐
│ LoginWindow          │  fork    │ QGuiApplication + QML   │
│ - 登录认证           │────────→ │ - RdpViewItem 渲染      │
│ - VM 列表管理        │  启动    │ - FreeRDP 连接线程      │
│ - 启动 qf-client     │          │ - 剪贴板/USB/音频       │
│ - 心跳保活           │          │ - 动态分辨率            │
└──────────────────────┘          └─────────────────────────┘
```

## 1. 系统要求

### 操作系统
- Debian (本书写时使用 Debian trixie/sid)

### 工具链
```
cmake >= 3.16
gcc / g++         (用于编译 FreeRDP + VDIClient)
clang / clang++   (用于编译 qf-client)
```

### 安装系统依赖

```bash
# 基础编译工具
sudo apt-get install -y cmake gcc g++ clang pkg-config

# Qt 6 (VDIClient: Widgets/Network; qf-client: QML/Quick)
sudo apt-get install -y qt6-base-dev qt6-declarative-dev

# FreeRDP 编译依赖
sudo apt-get install -y \
  libasound2-dev       \   # ALSA（FreeRDP 库编译依赖，qf-client 不直接使用）
  libpulse-dev         \   # PulseAudio 音频输入/输出 (rdpsnd + audin 主后端, 通过 PipeWire)
  libssl-dev           \   # TLS/SSL
  libusb-1.0-0-dev     \   # USB 重定向 (urbdrc)
  libv4l-dev           \   # 摄像头重定向 (rdpecam)
  libva-dev            \   # VAAPI (当前未启用)
  libavcodec-dev       \   # FFmpeg 编解码
  libswscale-dev       \   # FFmpeg 图像缩放
  libgbm-dev           \   # GBM (当前未使用，保留)
  libegl-dev           \   # EGL (当前未使用，保留)
  libgles-dev              # GLESv2（Debian trixie 上 libgles2-dev 已废弃，使用 libgles-dev）

# 日志
sudo apt-get install -y libspdlog-dev

# FreeRDP 附加功能
sudo apt-get install -y \
  libkrb5-dev          \   # Kerberos 认证 (可选)
  libcjson-dev         \   # JSON 解析
  libpcsclite-dev      \   # 智能卡
  libcups2-dev         \   # 打印机
  libfuse3-dev             # FUSE 3 驱动重定向（FreeRDP 3.28.0 需要 FUSE 3，FUSE 2 不适用）
```

## 2. 编译 FreeRDP 3.28.0

### 配置 & 编译

```bash
cd /home/kk/vdi-client/VDIClient/freerdp-3.28.0

# 清理旧构建（如有）
rm -rf build

# 配置
cmake -S . -B build \
  -DCMAKE_INSTALL_PREFIX="$PWD/install" \
  -DCMAKE_C_COMPILER=gcc \
  -DCMAKE_CXX_COMPILER=g++ \
  -DCMAKE_BUILD_TYPE=Release \
  -DWITH_ALSA=ON \
  -DWITH_PULSE=ON \
  -DWITH_FFMPEG=ON \
  -DWITH_SWSCALE=ON \
  -DWITH_VAAPI=OFF \
  -DWITH_VAAPI_H264_ENCODING=OFF \
  -DWITH_X11=OFF \
  -DWITH_WAYLAND=OFF \
  -DWITH_SERVER=OFF \
  -DWITH_CLIENT=OFF \
  -DWITH_CCACHE=OFF \
  -DWITH_SAMPLE=OFF \
  -DWITH_KRB5=OFF \
  -DWITH_CUPS=OFF \
  -DWITH_FUSE=ON \
  -DWITH_OSS=OFF \
  -DWITH_PCSC=OFF \
  -DWITH_UNICODE_BUILTIN=ON \
  -DWITH_VERBOSE_WINPR_ASSERT=ON \
  -DCHANNEL_RDPECAM=ON \
  -DCHANNEL_RDPECAM_CLIENT=ON
  -DCHANNEL_URBDRC=ON \
  -DCHANNEL_AINPUT=ON \
  -DCHANNEL_VIDEO=ON

# 编译（使用所有 CPU 核心）
cmake --build build -j$(nproc)

# 安装到 install/ 目录
cmake --install build
```

### 编译产物

```
install/
├── include/
│   └── freerdp3/          # FreeRDP 头文件
├── lib/
│   ├── libfreerdp3.so      # FreeRDP 核心库 (~3MB)
│   ├── libfreerdp-client3.so  # FreeRDP 客户端库 (~1MB)
│   ├── libwinpr3.so        # WinPR 跨平台层 (~1.5MB)
│   └── cmake/              # CMake 配置模块
├── bin/
│   ├── sfreerdp            # FreeRDP 服务器 (仅测试用)
│   ├── winpr-makecert
```

## 3. 编译 qf-client (QML + FreeRDP)

### 配置 & 编译

```bash
cd /home/kk/vdi-client/VDIClient/qfreerdp-master

# 清理旧构建（如有）
rm -rf build

# 配置（使用 clang 编译器）
cmake -DCMAKE_C_COMPILER=clang -DCMAKE_CXX_COMPILER=clang++ -B build .

# 编译
cmake --build build -j$(nproc)
```

### 编译说明
- CMake 通过 `list(PREPEND CMAKE_PREFIX_PATH ...)` 自动找到 `../freerdp-3.28.0/install` 下的 FreeRDP
- 使用 **clang** 编译器（已通过 Debian clang 19 验证）
- 使用 **C++20** 标准（qf-client 源码）和 **C23** 标准（cliprdr 等 C 源码）
- 渲染架构：**CPU 转存帧缓冲 + glTexSubImage2D 上传**（非 DMA-BUF 零拷贝）
- 音频架构：**rdpsnd（播放）+ audin（麦克风）均使用 FreeRDP 库自带的 PulseAudio 后端**，通过 PipeWire 实现。qf-client 无自编译音频后端代码（`rdpsnd-src/`、`audin-src/` 均已移除），音频完全由 FreeRDP 库管理。
- **重要：RUNPATH 问题** — 如果项目被移动或复制到新路径，必须**删除 build 目录重新编译**，否则运行时的 RUNPATH 仍指向旧路径，链接器会回退到系统 FreeRDP 库（可能不含 FFmpeg 支持），导致摄像头压缩失效。

### 编译产物

```
build/
└── qf-client              # 可执行文件 (~1.4MB, ELF 64-bit)
```

### 单独运行测试

```bash
cd /home/kk/vdi-client/VDIClient/qfreerdp-master

# 直接连接到远程桌面
./build/qf-client /v:192.168.1.90 /u:administrator /p:123456 /cert:ignore /drive:HOME,/home

# 或带日志输出
./build/qf-client /v:192.168.1.90 /u:administrator /p:123456 /cert:ignore /drive:HOME,/home &> /tmp/qf-client.log
```

## 4. 编译 VDIClient (Qt Widgets 登录管理)

### 配置 & 编译

```bash
cd /home/kk/vdi-client/VDIClient/vdi-client-linux

# 清理旧构建（如有）
rm -rf build

# 配置
cmake -B build .

# 编译
cmake --build build -j$(nproc)
```

### 部署 qf-client 到 VDIClient

**VDIClient 启动时会在自己的 `applicationDirPath()` 下查找 `bin/qf-client`**。
因此需要将 qf-client 可执行文件复制到 VDIClient 的构建输出目录：

```bash
# 将 qf-client 复制到 VDIClient 的 bin/ 目录
cp /home/kk/vdi-client/VDIClient/qfreerdp-master/build/qf-client \
   /home/kk/vdi-client/VDIClient/vdi-client-linux/build/bin/qf-client
```

VDIClient 的查找逻辑（`loginwindow.cpp:1436-1457`）：
1. 先查找 `applicationDirPath()/bin/qf-client`
2. 找不到则逐级向父目录查找 `bin/qf-client`
3. 还找不到则使用 `applicationDirPath()/bin/qf-client` 作为 fallback

### 运行 VDIClient

```bash
/home/kk/vdi-client/VDIClient/vdi-client-linux/build/bin/VDIClient
```

### VDIClient 命令行参数（无 — 在 UI 中输入）

VDIClient 启动后显示登录界面，在界面中输入：
- **服务器地址**：VDI 管理服务器 (如 `192.168.1.60`)
- **用户名 / 密码**：VDI 平台登录凭据
- 登录后自动获取 VM 列表，点击「连接」启动 qf-client RDP 会话

### VDIClient 编译产物

```
build/bin/
├── VDIClient              # 主程序 (~500KB)
└── qf-client              # 需要手动复制 (见上)
```

## 5. 命令行参数 (qf-client)

| 参数 | 说明 | 示例 |
|------|------|------|
| `/v:<ip>` | 远程服务器地址 | `/v:192.168.1.90` |
| `/u:<user>` | 用户名 | `/u:administrator` |
| `/p:<pass>` | 密码 | `/p:123456` |
| `/cert:ignore` | 忽略证书验证 | `/cert:ignore` |
| `/g:<host:port>` | RD Gateway 地址 | `/g:gateway.example.com:443` |
| `/drive:HOME,<path>` | 磁盘重定向 | `/drive:HOME,/home/kk` |
| `/clipboard` | 启用剪贴板重定向（默认禁用） | `/clipboard` |
| `/clipboard:direction-to:local` | 剪贴板仅定向到本地 | `/clipboard:direction-to:local` |
| `/usb:<id>` | USB 设备重定向 | `/usb:vid_1234_pid_5678` |
| `/usb:device:*` | USB 全部设备 | `/usb:device:*` |
| `/camera` | 启用摄像头重定向 | `/camera` |
| `/mic` | 启用麦克风重定向 | `/mic` |
| `-a` | 音频输出 (PulseAudio) | `-a` |

### 常用连接命令

```bash
# 最小命令
./build/qf-client /v:192.168.1.90 /u:administrator /p:123456 /cert:ignore

# 完整桌面体验（磁盘+剪贴板+USB+音频）
./build/qf-client /v:192.168.1.90 /u:administrator /p:123456 /cert:ignore \
  /drive:HOME,/home/kk /clipboard /usb:device:* -a

# 通过 RD Gateway
./build/qf-client /v:192.168.1.100 /u:user /p:pass /cert:ignore \
  /g:gateway.example.com:443 /drive:HOME,/home/kk /clipboard
```

## 6. 常见问题

### Q: qf-client 崩溃，exit code 11 (SIGSEGV)
- **排查步骤**:
  1. 检查是否偶发：同一连接重试一次，如果第二次成功则可能是竞态条件
  2. 检查日志中 `notify_window_resized: SKIP — null guard`：如果 QML 在 RDP 连接完成前触发 resize，说明启动时序问题，通常无害
  3. 检查日志中 `GFX_RESET with pending=0`：刚连接时的 GFX 同步，正常
  4. 检查是否有 `pulse->stream=(nil)` 警告：音频通道初始化失败
  5. 如持续崩溃，尝试移除 `/clipboard`、`/usb:` 等参数排除通道加载问题
- **已知根因**: 早期 DMA-BUF + VAAPI 路径偶发崩溃，当前 CPU staging buffer 架构已大幅降低崩溃率

### Q: 连接失败，日志提示 `ERRCONNECT_POST_CONNECT_FAILED`
- **原因**: 动态虚拟通道（drdynvc）加载插件失败，最常见是 rdpecam 或 audin 缺失
- **解决**: 确认 FreeRDP 编译时加了 `-DCHANNEL_RDPECAM=ON`、`-DCHANNEL_RDPECAM_CLIENT=ON` 和 `-DCHANNEL_AUDIN=ON`

### Q: 找不到 FreeRDP 头文件或库
- **原因**: CMake 无法通过 `CMAKE_PREFIX_PATH` 找到 freerdp 安装目录
- **解决**: 确认 `freerdp-3.28.0/install/` 目录存在，且 `qfreerdp-master/` 与 `freerdp-3.28.0/` 在同一父目录 (`VDIClient/`) 下

### Q: 编译时提示 `gbm/egl/glesv2 not found`
- **解决**: `sudo apt-get install libgbm-dev libegl-dev libgles2-dev`
- **说明**: 当前渲染架构不依赖 EGL/GBM，但仍需 GLESv2 用于 GL 纹理渲染

### Q: 窗口缩放后新区域黑屏
- **根因**: Qt 场景图在 QML 布局变化后缓存了旧的 `glViewport`
- **修复**: `RenderNode::render()` 中调用 `glViewport(0, 0, vpW, vpH)` 显式覆盖
- **关键文件**: `rdp-view-item.h` — `RenderNode` 类的 `render()` 方法

### Q: FPS 低（24fps 左右）
- **逐步排查**:
  1. 检查日志中 `rdpgfx_decode` 的间隔，如果 >33ms 说明服务器编码是瓶颈
  2. 检查 `updateGdiFrame` 间隔，如果远大于 decode 间隔，说明客户端 dispatch 有瓶颈
  3. VAAPI 当前关闭，解码为软件实现（~33ms/帧），如需提升可重新编译 FreeRDP 开启 `WITH_VAAPI=ON`
- **已知瓶颈**: 服务端 H.264 编码速率 + CPU 软件解码双重限制

### Q: 剪贴板子选项不生效 (`/clipboard:direction-to:*`)
- **原因**: 子选项不会自动启用 `FreeRDP_RedirectClipboard`
- **解决**: qf-client 已自动处理：CLI 解析后检测到子选项但 `RedirectClipboard` 仍为 FALSE 时自动设为 TRUE

### Q: 摄像头重定向带宽高达 50 Mbps（无压缩）
- **原因**: qf-client 运行时链接到了**不含 FFmpeg 支持**的 FreeRDP 库
- **排查**:
  1. 检查 qf-client 实际加载的 FreeRDP 库：`ldd build/qf-client | grep freerdp3`
  2. 确认路径指向 `freerdp-3.28.0/install/lib/` 而非 `/usr/local/lib/`
  3. 确认 FreeRDP 库编译了 FFmpeg 支持：`strings <path>/libfreerdp3.so | grep WITH_VIDEO_FFMPEG` 应输出 `WITH_VIDEO_FFMPEG=ON`
- **修复**: 删除 `build/` 目录后重新编译 qf-client（RUNPATH 会自动修正到正确的 FreeRDP 安装路径）
- **原理**: 不含 FFmpeg 时 `freerdp_video_conversion_supported()` 的 stub 实现只允许输入=输出的格式对，协商退化为 `(YUY2, YUY2)` 原始裸传 → 1280×720×2×30 ≈ 55 Mbps；含 FFmpeg 时协商为 `(YUY2, H264)` → libx264 压缩到 ~1-2 Mbps

### Q: 音频无声（日志提示 `pulse->stream=(nil)`）
- **原因**: PulseAudio 音频流初始化失败（可能是 PipeWire/PulseAudio 服务未运行）
- **解决**:
  1. 确认 PipeWire/PulseAudio 在运行：`pactl info` 或 `pw-cli info`
  2. 检查日志确认 rdpsnd 和 audin 通道是否成功加载（`WLOG_DEBUG` 级别下会输出音频通道初始化信息）
  3. 如系统无 PipeWire，移除 `-a` 参数禁用音频输出

### Q: `WITH_VERBOSE_WINPR_ASSERT=ON` 警告
- **原因**: FreeRDP 编译时开启了运行时断言检查
- **影响**: Release 模式下性能影响小，可忽略
- **移除**: 重新编译 FreeRDP 时加 `-DWITH_VERBOSE_WINPR_ASSERT=OFF`
